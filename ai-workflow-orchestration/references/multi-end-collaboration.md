# 多端协作与文档自动化

## 一、多端操作的核心价值

### 为什么需要多端？

| 端 | 适用场景 | 核心优势 |
|----|---------|----------|
| 桌面端 | 深度工作、复杂编辑 | 大屏、键盘快捷键、多窗口 |
| 网页端 | 跨设备访问、快速查看 | 无需安装、随时随地 |
| 移动端 | 碎片化查看、轻量操作 | 便携、即时响应 |

**设计原则**：同一个任务，在不同端有适配的交互方式，数据实时同步。

## 二、文档识别与处理

### 文档链接识别流程

```
用户在桌面端输入框粘贴文档链接
  → 系统自动识别链接类型（飞书/腾讯文档/Notion等）
  → 检查权限（用户是否已授权该平台）
  → 未授权：弹出授权流程
  → 已授权：解析文档内容
  → 展示文档名 + 摘要 + 操作选项
```

### 权限处理策略

**OAuth授权流程**：
1. 用户首次粘贴某平台链接
2. 系统检测未授权，弹出授权提示
3. 跳转到平台OAuth授权页
4. 用户确认授权，回调到系统
5. 保存token，后续自动使用

**权限不足的处理**：
- 明确提示："该文档权限不足，请申请'可编辑'权限后重试"
- 提供快捷申请入口（如支持）
- 降级方案：提供手动复制内容的方式

### 文档内容解析

**解析目标**：
- 提取标题、大纲、关键内容
- 识别文档结构（段落、列表、表格）
- 提取元数据（作者、修改时间、版本）

**解析结果展示**：
```
📄 文档：Q2产品规划
   最后修改：2026-06-03 | 作者：张三

   大纲：
   1. 背景与目标
   2. 核心功能规划
   3. 排期与里程碑
   ...

   [查看完整内容] [基于该文档生成任务] [补充内容到文档]
```

## 三、跨端任务同步

### 任务状态同步机制

```
桌面端创建任务
  → 写入云端数据库
  → 推送通知到移动端
  → 移动端拉取最新状态

移动端修改任务状态
  → 写入云端数据库
  → 推送通知到桌面端
  → 桌面端刷新界面
```

**冲突处理**：
- 最后写入优先（简单场景）
- 提示用户选择（复杂场景，如两端同时编辑）
- 操作日志可追溯（谁在什么时候改了什么）

### 移动端适配设计

**任务查看界面**：
- 列表视图：任务名、状态、截止时间
- 详情视图：描述、子任务、附件
- 快速操作：标记完成、修改状态、添加评论

**轻量编辑支持**：
- 修改任务状态（待办→进行中→已完成）
- 添加评论/备注
- 上传照片（如现场拍照记录）
- 不鼓励在移动端做复杂编辑（引导到桌面端）

## 四、定时任务与资讯收集

### 定时任务设计

**典型场景**：
- 每日AI行业新闻收集与推送
- 每周竞品动态追踪
- 品牌舆情监控周报
- 每日天气预报推送

**任务配置格式**：
```yaml
task:
  name: "AI行业新闻日报"
  schedule: "0 9 * * *"  # 每天9点
  action:
    type: "collect_news"
    params:
      keywords: ["AI", "大模型", "AIGC"]
      max_results: 10
      include_links: true
  delivery:
    channel: "feishu"  # 推送渠道
    target: "我的飞书消息"
```

### 资讯收集工作流

```
定时触发（每天9点）
  → 调用搜索API收集最新资讯
  → AI筛选：去重、评分相关性
  → 生成结构化日报（标题、摘要、链接）
  → 推送到指定渠道（飞书/邮件/Webhook）
  → 记录执行日志（成功/失败、收集数量）
```

**日报格式示例**：
```
📰 AI行业新闻日报 | 2026-06-04

1. [OpenAI发布GPT-5预览版](https://example.com/1)
   摘要：OpenAI今日发布了GPT-5的开发者预览版...
   相关度：⭐⭐⭐⭐⭐

2. [谷歌Gemini 2.0支持实时语音](https://example.com/2)
   摘要：谷歌宣布Gemini 2.0正式支持实时语音交互...
   相关度：⭐⭐⭐⭐

---
共收集12条，筛选出5条高相关资讯
[查看完整报告] [调整关键词]
```

### 任务模板库

**内置模板**：
| 模板名 | 适用场景 | 默认频率 |
|--------|---------|----------|
| 品牌舆情监控 | 监控品牌关键词在社交媒体的出现 | 每日 |
| 竞品动态追踪 | 追踪竞争对手的产品更新和新闻 | 每周 |
| 行业政策速递 | 收集行业相关政策和法规更新 | 每周 |
| AI技术前沿 | 收集AI/ML领域的最新研究和技术 | 每日 |
| 用户反馈汇总 | 汇总各渠道用户反馈并分类 | 每周 |

**自定义模板**：
用户可保存自己的任务配置为模板，方便复用。

## 五、文档自动化编辑

### 内容生成与回写

**工作流**：
```
AI基于文档内容生成补充内容
  → 展示预览（高亮新增/修改部分）
  → 用户确认或修改
  → 回写到原文档（或创建新版本）
  → 记录修改历史和理由
```

**回写策略**：
- 追加模式：在文档末尾追加新内容
- 插入模式：在指定位置插入新内容
- 更新模式：更新文档中特定章节
- 新建模式：创建新文档，保留原文档不变

### 变更追踪

**修改记录格式**：
```markdown
> [!NOTE]
> 由AI助手于2026-06-04 15:30补充
> 理由：基于活动流程讨论，补充详细执行步骤

## 活动执行步骤（新增）

1. 前期准备（活动前7天）...
2. 现场执行（活动当天）...
```

## 六、常见问题解答

### Q1：文档解析失败怎么办？
**A**：1）检查链接权限；2）确认文档格式是否支持；3）尝试重新粘贴链接；4）如果仍然失败，提供手动复制内容的降级方案。

### Q2：定时任务没有按时推送怎么办？
**A**：检查：1）任务是否启用；2）推送渠道配置是否正确；3）系统时间是否准确；4）查看任务执行日志。

### Q3：移动端能完全替代桌面端吗？
**A**：不建议。移动端设计为轻量查看和快速操作，复杂编辑和配置仍应在桌面端完成。

### Q4：如何避免资讯收集重复推送？
**A**：系统在收集时会做去重处理（基于标题/URL相似度）。如果仍然有重复，可以调整去重阈值或手动标记"不感兴趣"。
