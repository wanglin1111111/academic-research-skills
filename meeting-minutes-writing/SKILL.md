# 会议总结（纪要）写作指南

本技能帮助将会议对话提炼为高质量会议纪要，区别于流水账式的会议记录，纪要是对会议成果的再加工和归纳，旨在传达会议精神与推动执行。

## 核心理念

会议纪要的灵魂是**"纪要"而非"记录"**：
- 会议记录 = 事务文书，如实记载原始情况
- 会议纪要 = 正式公文，提炼共识、结论和行动

**好纪要的标准**：1小时会议 → 2分钟读完核心 → 清晰指导后续行动

## 四大功能模块

### 1. 标准化结构与模板
**触发词**：会议纪模板、纪要格式、会议总结怎么写、标准结构

当用户需要会议纪要模板或结构调整时，引用 `references/standard-template.md`。

**核心结构（5个模块）**：
1. 会议概况（头部信息）
2. 核心内容与讨论要点
3. 会议决议与行动项（SMART表格）
4. 遗留问题与待解决事项
5. 后续安排

### 2. 写作原则与三个"必须"
**触发词**：写作原则、三个必须、五个什么、纪要权威性

当用户询问纪要写作规范、人称用法、如何确保执行力时，引用 `references/writing-principles.md`。

**三个"必须"**：
- 必须如实反映会议内容（纪实性）
- 必须围绕会议成果提炼（非流水账）
- 必须反映集体意志（第三人称："会议认为/指出/决定/要求"）

**五个"什么"（执行力保障）**：
- 什么时间商量的？
- 我们要做什么事情？
- 由什么人来负责？
- 什么时间完成？
- 要达到什么效果？

### 3. 分类型会议侧重点
**触发词**：项目例会纪要、战略会议纪要、头脑风暴纪要、客户会议纪要

当用户说明会议类型，需要针对性写作指导时，引用 `references/meeting-types.md`。

**四类会议侧重点**：
| 会议类型 | 侧重点 | 精简程度 |
|----------|---------|-----------|
| 项目例会/站会 | 进度/计划/阻碍 + 行动项/风险 | 极度精简 |
| 战略规划会 | 背景数据/分析选项/决策理由 | 详细记录逻辑 |
| 头脑风暴/创意会 | 保留所有想法，按主题分类 | 不急于评估 |
| 客户/外部会议 | 准确谨慎，只记录共识和承诺 | 准确为先，避免内部信息泄露 |

### 4. 常见误区与避坑指南
**触发词**：纪要误区、避坑、待办模糊、分发延迟

当用户提及纪要质量问题、执行跟踪、分发时效时，引用 `references/common-pitfalls.md`。

**四大误区**：
1. 写成"逐字稿" → 灵魂在提炼，不在记录
2. 待办事项模糊 → "尽快跟进"不如"张三6月11日前完成XX"
3. 缺少上下文 → 未参会者也能读懂
4. 分发严重延迟 → 最佳时效：会议结束后24小时内

## 工作流程

### 步骤1：信息收集
- 获取会议录音/逐字稿/笔记
- 确认会议类型（决定侧重点）
- 明确参会人员及缺席情况

### 步骤2：内容提炼
- 按议题归纳，非按时间顺序
- 提炼讨论要点（核心观点、质疑、数据）
- 明确主要结论（统一认识、明确方向）

### 步骤3：行动项表格化
- 遵循SMART原则
- 表格呈现：任务描述 | 负责人 | 截止日期 | 优先级 | 验收标准

### 步骤4：结构组装
- 套用5模块标准结构
- 检查三个"必须"是否达标
- 检查五个"什么"是否都有答案

### 步骤5：审核分发
- 发送给参会者确认（特别是客户会议）
- 24小时内发出（趁记忆新鲜）
- 跟踪行动项落实，形成闭环

## 重要提醒

1. **纪实性是第一原则**：不能凭空捏造或主观臆断
2. **第三人称表述**：用"会议认为/指出/决定/要求"，不用"我认为/建议"
3. **行动项必须SMART**：具体、可衡量、可达成、相关、有截止日期
4. **纪要不是结束**：签发后持续跟踪待办落实，形成工作闭环

## 使用建议

1. 先判断会议类型，再选择对应的侧重点和精简程度
2. 行动项表格是纪要最核心的部分，务必具体可执行
3. 善用AI辅助工具做初步转录，但人工提炼不可替代
4. 建立个人素材库，对比领导修改版本，快速提升写作水平

## 限制说明

本技能提供会议纪要的写作框架和原则，具体内容需结合实际会议情况填充。涉及对外发文（如客户会议纪要），建议经过法务或相关负责人审核后再发出。
