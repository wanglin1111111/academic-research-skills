# 会议总结写作原则与三个"必须"

## 一、会议记录 vs 会议纪要

| 维度 | 会议记录 | 会议纪要 |
|------|----------|----------|
| 性质 | 事务文书 | 正式公文 |
| 内容 | 如实记载原始情况 | 提炼归纳，再加工 |
| 目的 | 留档备查 | 传达精神、推动执行 |
| 写作方式 | 逐字/逐事记录 | 提炼观点、结论、行动 |
| 读者 | 未来查阅的人 | 所有需要执行/知情的人 |

**核心区别**：会议记录是"摄像机"，会议纪要是的"编辑器"。

---

## 二、三个"必须"（奠定纪要权威性）

### 必须1：必须如实反映会议内容

**含义**：纪要源于会议，不能凭空捏造或主观臆断。

**错误做法**：
```
✗ 会议没有讨论预算，但记录人自己加上了预算建议
✗ "会议决定全员降薪10%"，但实际只讨论了可行性
✗ 把个人观点写成"会议认为"
```

**正确做法**：
```
✓ 只写会议实际讨论过的内容
✓ 结论区分"已决定"和"待确认"
✓ 有疑问时找参会者核对，不自行补全
```

---

### 必须2：必须围绕会议成果进行提炼

**含义**：切忌写成"流水账"，重点介绍共识、结论和下一步行动。

**流水账写法（✗）**：
```
09:00 会议开始，张三主持
09:05 李四汇报了上周进度，说后端完成了60%
09:12 王五问李四前端能不能配合，李四说可以
09:20 张三说要注意质量...
（逐分钟记录，读完不知道结论是什么）
```

**成果提炼写法（✓）**：
```
### 议题一：Q2功能开发进度

**讨论要点**：
- 后端完成60%，前端进度滞后约3天
- 李四确认可配合前端联调，预计追回1天

**主要结论**：
- Q2功能开发整体进度可控，联调后预计按期交付
- 风险点：前端资源仍偏紧，持续关注
```

---

### 必须3：必须反映集体意志

**含义**：通常使用第三人称，以"会议"作为表述主体。

**人称使用规范**：

| 表述 | 适用场景 | 示例 |
|------|---------|------|
| "会议认为" | 对某个问题达成统一认识 | "会议认为，当前优先级应为稳定性优于新功能" |
| "会议指出" | 点出存在的问题或风险 | "会议指出，前端资源瓶颈可能影响上线时间" |
| "会议决定" | 正式决议，已形成结论 | "会议决定，增加1名外包前端支持" |
| "会议要求" | 对执行提出要求 | "会议要求，所有功能必须在6月15日前提测" |
| "会议同意" | 批准某方案或申请 | "会议同意，预算增加5万元用于外包" |

**第一人称禁用场景**：
```
✗ "我认为我们应该..."          →  ✓ "会议认为..."
✗ "我建议各部门在周五前..."   →  ✓ "会议要求各部门在周五前..."
✗ "我已经安排了张三去跟进"    →  ✓ "会议确定由张三负责跟进"
```

**例外情况**：在"会议概况"的"记录人"字段中，可以用第一人称署名，但正文保持第三人称。

## 三、五个"什么"（确保可执行性）

好纪要的核心价值：**未来的行动指南**。

把以下5个问题答案以清单形式呈现，纪要就清晰可落地：

```
1. 什么时间商量的？   → 会议时间（头部信息）
2. 我们要做什么事情？   → 会议决议（行动项表格）
3. 由什么人来负责？   → 负责人（行动项表格）
4. 什么时间完成？      → 截止日期（行动项表格）
5. 要达到什么效果？   → 验收标准（行动项表格）
```

**应用示例**（行动项表格中体现五个"什么"）：

| 任务描述（做什么） | 负责人（谁来做） | 截止日期（何时完成） | 验收标准（什么效果） |
|---------|---------|----------|----------|
| 完成权限管理模块技术方案 | 李四 | 2026-06-07 | 输出技术方案文档且评审通过 |
| 启动外包人员招聘流程 | 张三 | 2026-06-06 | 至少面试3家供应商并提交比选报告 |

## 四、写作注意事项

### 表述准确度

**层级用词区别**：
```
"会议认为"   → 统一认识，未必有行动
"会议决定"   → 已形成决议，必须执行
"会议建议"   → 非强制性，仅供参考
"会议要求"   → 有明确执行要求
```

### 结论模糊的处置

**当会议未形成明确结论时**：
```
✗ 错误：假装已有结论，自行补充
✓ 正确：如实记录"未达成共识，待XX前进一步调研后确定"
```

**标注方式**：
```
**主要结论（待确认）**：
- 方案A和方案B各有优劣，李四将在下周三前完成成本对比分析后再次评审
- 本次会议暂未确定最终方案
```

## 五、常见问题解答

### Q1：参会人要求把他的发言写得更"正面"怎么办？
**A**：可以优化表述方式，但不能改变事实结论。建议写法："会议讨论了XX问题，[李四]提出了方案A，[王五]提出了方案B，会议决定进一步调研后决策"——呈现讨论过程，不偏向任何一方。

### Q2：会议上吵得很厉害，没形成结论，纪要怎么写？
**A**：如实记录讨论过程和未决事项，在"遗留问题与待解决事项"模块中详细列出，并明确跟进人和解决路径。没结论也是结论。

### Q3：可以事后补充会议上没有说的内容吗？
**A**：不可以。如果会后有了新的决定，应发"会议纪要补充通知"，而非直接改原纪要。纪要必须如实反映会议当时的内容。
