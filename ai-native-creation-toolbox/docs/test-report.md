## 三、发现的问题

### 问题1：提示词太长，AI可能漏掉部分内容
- **现象**：模板四原版把三个阶段放在一个提示词里
- **后果**：AI可能只执行了部分（比如只做第一阶段，忽略第二、三阶段）
- **根因**：提示词过长，超出AI的"任务边界感知"能力
- **严重程度**：⚠️ 中等（影响执行完整性）

### 问题2：第二阶段"生成MVP"指令不够具体
- **现象**：提示词说"生成可运行版本"，但没说明：
  - 如果是工具 → 生成什么语言/框架的代码？
  - 如果是产品 → 生成文字描述还是界面原型？
  - 如果是游戏 → 生成什么平台的可运行版本？
- **后果**：AI可能生成不符合预期的内容（比如你要代码，它给了文字描述）
- **根因**：缺少"MVP形式选择"的前置决策
- **严重程度**：⚠️ 中等（需要反复迭代才能得到想要的输出）

### 问题3：第三阶段评估太温和
- **现象**："自我评估问卷"容易给自己打高分（人性使然）
- **后果**：可能继续深入一个实际上不值得做的方向
- **根因**：自我评估的"自我服务偏差"（Self-serving bias）
- **严重程度**：🚨 高（可能导致浪费时间在不值得的方向上）

## 四、改进方案与实施

### 改进1：拆分模板四为三个独立提示词
- **方案**：把原版模板四拆成：
  - **4.1**（第一阶段：想法澄清）
  - **4.2**（第二阶段：MVP实现）
  - **4.3**（第三阶段：挑战者评估）
- **实施**：✅ 已完成（写入技能文件）
- **效果**：每个提示词更短、更聚焦，AI执行更完整

### 改进2：增加MVP形式选择决策树
- **方案**：在4.2提示词开头，让AI先判断"这个MVP最适合用哪种形式实现？"
  - (a) 可运行代码（工具/脚本/自动化）
  - (b) 原型描述 + 界面草图（产品/功能/界面）
  - (c) 详细规格说明（内容/服务/方法论）
- **实施**：✅ 已完成（写入技能文件）
- **效果**：AI生成的内容更符合预期，减少迭代次数

### 改进3：把"自我评估"改成"挑战者评估"
- **方案**：在4.3提示词中，让AI扮演"严厉的挑战者"，提出4个尖锐问题：
  1. 关于价值："为什么用户会用你的而不是已有的解决方案？你的差异化到底是什么？"
  2. 关于壁垒："你的这个MVP，有什么是别人复制不了的？如果没有，你打算怎么建立壁垒？"
  3. 关于执行："你真的会在未来3天内继续改进它吗？如果答案是'看情况'，说明你热情不够"
  4. 关于市场："你愿意为这个MVP付多少钱？如果答案是0或接近0，说明它对你来说价值不够"
- **实施**：✅ 已完成（写入技能文件）
- **效果**：评估更严格，减少"虚假阳性"（不值得做的方向被判断为值得做）
