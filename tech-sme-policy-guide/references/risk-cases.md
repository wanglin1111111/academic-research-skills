# 科技型中小企业申报风险与案例汇编

## 一、申报风险识别

### 风险1：中介代报风险

**政策依据**：
- 工信部明确通知：企业"不通过第三方中介机构申报"
- 违规操作可能导致申报失败或被追责

**风险表现**：
1. 中介机构包装材料，数据不真实
2. 企业不了解申报内容，无法应对核查
3. 中介机构承诺"包过"，实际违规操作
4. 被抽查时企业无法提供真实证明材料

**案例1：中介承诺"包过"导致企业被撤销编号**
- **案情**：某科技咨询公司承诺"包过"，收取高额费用为企业包装申报材料。企业获得入库编号后，在集中抽查中被发现材料造假。
- **后果**：撤销入库编号，追回已享受的税收优惠，企业被列入科研失信名单。
- **教训**：企业应自主申报，确保材料真实。

**规避建议**：
1. 企业自主申报，不委托中介机构
2. 如需要专业指导，可咨询正规咨询机构，但最终由企业自己申报
3. 申报材料必须由企业自己准备，确保真实性

### 风险2：数据不一致风险

**政策依据**：
- 申报数据（尤其是研发费用）必须与税务申报表（A107012）保持一致
- 系统会自动进行比对核查

**风险表现**：
1. 研发费用辅助账与税务报表不一致
2. 销售收入、资产总额等财务数据异常波动
3. 科技人员数量与社保缴纳人数不符
4. 知识产权数量与实际情况不符

**案例2：研发费用与税务报表不一致被撤销编号**
- **案情**：某企业申报时填报研发费用100万元，但税务报表（A107012表）显示研发费用仅60万元。系统比对发现差异，要求企业说明。
- **后果**：企业无法提供合理解释，被撤销入库编号。
- **教训**：申报数据必须与税务报表保持一致。

**规避建议**：
1. 申报前先核对税务报表数据
2. 确保研发费用辅助账与税务报表一致
3. 财务数据避免异常波动，如有波动需提供合理说明
4. 科技人员数量以社保缴纳人数为准

### 风险3：科技属性不足风险

**政策依据**：
- 明确严禁将单纯从事商贸流通、简单组装加工、无自主研发活动的企业包装入库

**风险表现**：
1. 企业实际从事商贸流通，无研发活动
2. 企业仅从事简单组装加工，无自主研发
3. 研发费用归集不合理，将非研发费用计入
4. 科技人员实际不从事研发活动

**案例3：商贸企业包装入库被抽查发现**
- **案情**：某商贸企业，实际从事电子产品销售，无研发活动。为申报科小，临时归集"研发费用"，并将销售人员计入"科技人员"。实地核查被发现。
- **后果**：撤销入库编号，企业被通报批评。
- **教训**：企业必须具备真实研发活动。

**规避建议**：
1. 企业应有真实的研发活动
2. 研发费用归集应合理，有凭证支持
3. 科技人员应实际从事研发活动
4. 保留研发活动证明材料（立项报告、研发日志、测试记录等）

### 风险4：知识产权风险

**政策依据**：
- 知识产权数量为0的企业需实地核查
- 知识产权应与主营业务相关

**风险表现**：
1. 知识产权数量为0，无法合理解释
2. 知识产权与主营业务无关，被视为包装
3. 知识产权已过期或无效
4. 知识产权权属不清，存在纠纷

**案例4：知识产权与主营业务无关被要求说明**
- **案情**：某制造企业，主营业务为机械加工。为申报科小，购买了一项与主营业务无关的软件著作权。审核时被发现，要求说明相关性。
- **后果**：企业无法说明，被要求撤回申报。
- **教训**：知识产权应与主营业务相关。

**规避建议**：
1. 提前布局与主营业务相关的知识产权
2. 确保知识产权在有效期内
3. 确保知识产权权属清晰
4. 如知识产权数量为0，应准备充分的研发活动证明材料

### 风险5：突击申报风险

**政策依据**：
- 每个批次企业公告入库后立即进行集中抽查
- 抽查比例不低于5%

**风险表现**：
1. 企业在申报前临时归集研发费用
2. 临时增加科技人员数量
3. 临时申请知识产权
4. 财务数据在申报前异常波动

**案例5：临时归集研发费用被抽查发现**
- **案情**：某企业为申报科小，在申报前1个月临时归集研发费用50万元。集中抽查时被发现费用归集不合理，缺乏凭证支持。
- **后果**：被要求补正材料，延迟入库时间。
- **教训**：研发费用归集应常态化，不能临时突击。

**规避建议**：
1. 研发费用归集应常态化，有完整的辅助账
2. 科技人员统计应基于实际情况
3. 知识产权应提前布局，不能临时突击
4. 财务数据应保持稳定，避免异常波动

### 风险6：编号有效期风险

**政策依据**：
- 入库登记编号仅当年有效（截止到当年12月31日）
- 次年需重新申请

**风险表现**：
1. 企业误以为编号长期有效
2. 次年未及时重新申报
3. 编号过期后仍享受优惠政策

**案例6：编号过期仍享受税收优惠被查处**
- **案情**：某企业2025年获得科小编号，2026年未重新申报，但仍在享受研发费用加计扣除政策。税务部门核查时发现。
- **后果**：追回已享受的税收优惠，企业被处罚。
- **教训**：科小编号仅当年有效，次年需重新申报。

**规避建议**：
1. 每年按时重新申报
2. 关注编号有效期，及时续报
3. 编号过期后不再享受相关优惠政策

## 二、审核不通过常见原因

### 原因1：科技人员占比<10%，得0分

**解决方案**：
1. 增加科技人员数量
2. 将从事研发相关工作的人员计入科技人员
3. 确保科技人员统计口径准确

### 原因2：研发费用占比<2%，得0分

**解决方案**：
1. 增加研发费用归集
2. 规范研发费用辅助账
3. 确保研发费用应归尽归

### 原因3：无知识产权，得0分

**解决方案**：
1. 申请知识产权（发明专利、实用新型、软件著作权等）
2. 购买与主营业务相关的知识产权
3. 确保知识产权在有效期内

### 原因4：数据不一致，被系统比对发现

**解决方案**：
1. 确保申报数据与税务报表一致
2. 核对财务数据，避免异常波动
3. 提供数据差异的合理解释

### 原因5：材料不完整，被退回补正

**解决方案**：
1. 按照要求准备完整材料
2. 上传清晰的附件材料
3. 及时补正退回的材料

## 三、风险规避 checklist

### 申报前自查

- [ ] 企业是否符合基础准入条件
- [ ] 科技人员占比是否≥10%
- [ ] 研发费用占比是否≥2%
- [ ] 是否拥有有效的知识产权
- [ ] 申报数据是否与税务报表一致
- [ ] 是否建立了研发费用辅助账
- [ ] 企业是否从事真实的研发活动
- [ ] 知识产权是否与主营业务相关

### 申报中检查

- [ ] 所有必填项是否已填写
- [ ] 上传的附件材料是否清晰完整
- [ ] 数据逻辑是否合理
- [ ] 是否保存了申报草稿

### 申报后跟踪

- [ ] 是否及时关注审核状态
- [ ] 如被退回，是否及时补正
- [ ] 如被实地核查，是否积极配合
- [ ] 是否关注公示情况
- [ ] 是否及时获取入库编号

### 入库后维护

- [ ] 是否保留完整的申报材料备查
- [ ] 是否规范享受优惠政策
- [ ] 是否准备应对集中抽查
- [ ] 是否关注编号有效期，及时续报

## 四、合规建议

### 建议1：建立常态化研发管理体系

1. 设立研发部门，明确研发人员
2. 建立研发费用辅助账，规范归集研发费用
3. 制定研发项目立项、实施、验收流程
4. 保留研发活动完整证明材料

### 建议2：提前布局知识产权

1. 根据主营业务方向，提前申请知识产权
2. 确保知识产权与主营业务相关
3. 维护知识产权有效性，及时缴纳年费
4. 避免临时突击申请

### 建议3：规范财务核算

1. 按照研发费用加计扣除政策要求，规范归集研发费用
2. 确保研发费用辅助账与税务报表一致
3. 避免财务数据异常波动
4. 保留完整的财务凭证

### 建议4：自主申报，确保真实

1. 企业自主申报，不委托中介机构
2. 申报材料由企业自己准备，确保真实性
3. 了解申报内容，能够应对核查
4. 如实填写融资需求调研等问题

## 五、常见问题解答

### Q1：被抽查了怎么办？
**A**：积极配合，提供相关证明材料。确保材料真实准确。

### Q2：申报失败还能再次申报吗？
**A**：可以。查明失败原因，补充完善后再次申报。

### Q3：入库后被撤销编号怎么办？
**A**：查明原因，整改后下一年度重新申报。

### Q4：可以同时申报高企和科小吗？
**A**：可以。高企可以直接走"直通车"条件入库，无需评分。

### Q5：分公司可以申报吗？
**A**：分公司不具有法人资格，不能独立申报。应由具有法人资格的总公司申报。
