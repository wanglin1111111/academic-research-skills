# 企业AI平台建设与管理指南

本技能帮助企业从零到一构建AI平台，强调"场景牵引、技术驱动、制度保障"的三维协同发展路径，覆盖战略定位→架构设计→数据治理→MLOps→FinOps→安全合规的完整闭环。

## 核心理念

AI平台建设的核心原则：
1. **场景牵引** — 回归业务本质，以实际场景驱动建设
2. **技术驱动** — 分层解耦、云原生、可扩展
3. **制度保障** — 全生命周期合规、安全可审计

避免两个极端：
- ❌ 盲目追求技术领先（"我们也要有大模型"）
- ❌ 数字形式主义（"上了系统就是数字化转型"）

## 七大功能模块

### 1. 战略定位与场景选择
**触发词**：AI平台战略、场景选择、业务对齐、OKR、数字化转型

当用户需要明确AI平台战略定位或选择落地场景时，引用 `references/strategic-positioning.md`。

**核心方法**：
- 选择"需求高、见效快"的场景作为切入点
- 将业务目标转化为可衡量指标（准确率、成本、合规性）
- 设定季度OKR确保平台价值与业务价值对齐

### 2. 分层架构设计
**触发词**：分层架构、云原生、Kubernetes、服务网格、MLOps架构

当用户需要设计AI平台架构或评估技术选型时，引用 `references/architecture-design.md`。

**核心架构（7层）**：
- 数据层 → 特征层 → 训练评估层 → 模型管理层 → 推理服务层 → 监控治理层 → AI应用层
- 云原生底座：Kubernetes + 服务网格

### 3. 数据治理框架
**触发词**：数据治理、合规框架、数据血缘、脱敏、GDPR、个人信息保护

当用户讨论数据治理、隐私合规或数据安全时，引用 `references/data-governance.md`。

**核心框架**：
- 全生命周期：采集→标注→训练→评估→部署→归档
- 合规三原则：脱敏、匿名化、最小化采集
- 安全策略：最小权限 + 加密存储 + 审计日志

### 4. MLOps流水线
**触发词**：MLOps、CI/CD流水线、模型注册、版本管理、A/B测试、监控

当用户需要设计MLOps流程或优化模型上线效率时，引用 `references/mlops-pipeline.md`。

**核心能力**：
- 数据准备→训练→评估→部署的CI/CD流水线
- 模型注册中心统一管理版本、标签、审批状态
- 实时监控：数据漂移、模型退化、推理延迟

### 5. FinOps成本优化
**触发词**：成本优化、FinOps、多云、算力调度、竞价实例、省级统筹

当用户关心AI平台成本控制或资源调度时，引用 `references/finops-optimization.md`。

**核心策略**：
- 精细化编排：训练重吞吐、推理重延迟
- 多云混合：本地合规+海外前沿能力
- 单位价值成本指标（如每千字生成成本）

### 6. 安全合规体系
**触发词**：AI安全、合规要求、幻觉问题、算法歧视、内容过滤、审核机制

当用户讨论AI安全风险或合规要求时，引用 `references/security-compliance.md`。

**核心防线**：
- 管理风险：组织架构+流程制度
- 技术风险：数据安全+模型幻觉+算法歧视
- 制度监管：算法备案+安全评估+人工审核兜底

### 7. 场景落地指南
**触发词**：AI落地场景、政务AI、金融AI、智慧城市、行业应用

当用户需要了解AI平台在不同行业的应用案例或落地路径时，引用 `references/scenario-playbook.md`。

**典型场景**：
- 政务：智能问答、政策直达、辅助决策
- 金融：智能投研、精准营销、智能风控
- 企业：智慧办公、数据分析预测、异常预警
- 城市：智慧城管、智慧农业、环境监控

## 工作流程

### 阶段1：战略规划（1-2月）
1. 调研业务场景与痛点
2. 确定AI平台战略定位
3. 选择"切入点"场景（需求高、见效快）
4. 设定OKR与成功指标

### 阶段2：架构设计（2-3月）
1. 设计分层架构
2. 选型技术栈（云原生底座）
3. 设计MLOps流水线框架
4. 制定数据治理规范

### 阶段3：建设实施（3-6月）
1. 搭建基础设施（Kubernetes+GPU集群）
2. 部署MLOps流水线
3. 落地数据治理平台
4. 实现首批场景应用

### 阶段4：运营优化（持续）
1. 监控平台运行指标
2. 持续优化成本
3. 迭代场景覆盖
4. 完善安全合规

## 重要提醒

1. **场景优先于技术**：不要先选技术再找场景，要先找场景再选技术
2. **合规是前提**：数据治理和安全合规必须在平台建设之初就规划，而非事后补救
3. **成本要量化**：每个场景都要计算"单位价值成本"，用业务产出衡量ROI
4. **MLOps是基础**：没有体系化MLOps，模型上线就是"一锤子买卖"，无法持续优化

## 使用建议

1. 先明确战略定位（阶段1），再设计架构（阶段2），不要跳步
2. 每个场景落地前，先计算ROI和成功指标，避免"为了AI而AI"
3. 数据治理和安全合规的设计要从第一天开始，不能等平台建成后再补
4. MLOps流水线的完整程度决定了后续运营效率，初期投入值得

## 限制说明

本技能提供企业AI平台建设的通用方法论，具体技术选型和架构设计需结合企业实际情况、行业合规要求和团队能力综合考虑。涉及金融、医疗等强监管行业时，建议同步咨询行业合规专家。
