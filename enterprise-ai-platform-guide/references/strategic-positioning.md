# 战略定位与场景选择

## 一、AI平台战略定位原则

### 两个极端要避免

| 极端 | 表现 | 后果 |
|------|------|------|
| 技术驱动型 | "我们也要部署大模型" | 建了不用，浪费投资 |
| 形式主义型 | "上了系统就是数字化转型" | 系统成为摆设，无业务价值 |

### 正确姿势：场景牵引

**核心逻辑**：
```
业务痛点 → 识别高价值场景 → 选择技术方案 → 验证价值 → 迭代扩展
```

**"场景牵引"的内涵**：
- 从业务痛点出发，而非从技术能力出发
- 优先选择"需求高、见效快"的场景作为切入点
- 避免贪大求全，追求小步快跑、快速验证

---

## 二、场景选择框架

### 场景筛选矩阵

| 维度 | 高价值 | 低价值 |
|------|--------|--------|
| **业务重要性** | 核心业务环节 | 边缘辅助环节 |
| **见效速度** | 1-3月可见成果 | 半年以上才见效 |
| **数据成熟度** | 数据积累完整 | 数据需大量治理 |
| **技术可行性** | 现有技术可支持 | 需突破性新技术 |
| **合规风险** | 低风险场景 | 强监管场景（初期） |

### 推荐的"切入点"场景特征

1. **高频刚需**：业务中使用频率高，痛点明确
2. **数据基础好**：已有结构化数据积累
3. **容错空间**：允许AI辅助，不要求100%准确
4. **易于量化**：效果可用准确率、成本、时间等指标衡量

## 三、不同行业的场景选择

### 政务领域

**优先场景**：
| 场景 | 特点 | 切入价值 |
|------|------|----------|
| 智能问答 | 高频、政策性强 | 减轻窗口压力 |
| 政策直达 | 政策查找困难 | 提升服务体验 |
| 辅助决策 | 数据分析需求 | 提升决策效率 |
| 智能监测 | 实时预警需求 | 降低人力成本 |

**避坑提示**：
- 避免直接面向公众的高风险决策场景
- 强调"辅助"定位，人工审核兜底

### 企业领域

**优先场景**：
| 场景 | 业务指标 | 价值量化方式 |
|------|----------|--------------|
| 客服质检 | 人工审核覆盖率 | 从10%→100% |
| 文档生成 | 撰写时间 | 从2小时→10分钟 |
| 供应链预测 | 库存周转率 | 提升X% |
| 智能办公 | 会议纪要效率 | 从2小时→5分钟 |

**OKR设定示例**：
```
O（目标）：AI平台赋能业务效率提升
KR1（关键结果）：客服质检覆盖率从10%提升至100%
KR2：文档生成平均耗时从2小时缩短至15分钟
KR3：供应链预测准确率提升15%
```

### 金融行业

**转型路径**：
```
第一阶段："效率工具"（短期1-2年）
  - 智能投研：研报摘要生成
  - 精准营销：用户行为分析
  - 运营自动化：智能录单

第二阶段："战略能力"（中期3-5年）
  - 智能风控：实时风险预警
  - 个性化投顾：千人千面
  - 决策支持：智能分析与建议
```

## 四、业务目标量化方法

### 指标体系设计

**三层指标**：
```
┌─────────────────────────────────────────┐
│  业务层指标（业务部门关注）              │
│  营收、成本、客户满意度、NPS             │
├─────────────────────────────────────────┤
│  平台层指标（IT/数据团队关注）           │
│  准确率、响应时间、模型调用量            │
├─────────────────────────────────────────┤
│  技术层指标（AI团队关注）                │
│  模型性能、漂移率、GPU利用率            │
└─────────────────────────────────────────┘
```

### 量化公式

| 场景 | 量化指标 | 计算方式 |
|------|----------|----------|
| 智能问答 | 解决率 | 成功解答数 / 总提问数 |
| 文档生成 | 采用率 | AI生成被采纳数 / 总生成数 |
| 风险预测 | 准确率 | 预测正确数 / 总预测数 |
| 成本优化 | 节省金额 | 人工成本 - AI成本 |

## 五、OKR设定与跟踪

### OKR模板

```yaml
# 季度OKR示例：AI平台赋能业务

## O（目标）
AI平台在Q2赋能至少3个核心业务场景，显著提升效率

## KR（关键结果）
- KR1：客服质检覆盖率达到100%，人工抽检时间减少80%
- KR2：文档生成采用率达到60%，平均耗时<15分钟
- KR3：供应链预测准确率提升15%，库存周转率提升10%
- KR4：至少5个业务部门完成AI平台接入培训
```

### 跟踪机制

| 周期 | 内容 | 参与者 |
|------|------|--------|
| 周 | 模型指标跟踪 | AI团队 |
| 双周 | 场景进度同步 | AI+业务 |
| 月 | OKR进度评审 | 高层 |
| 季度 | 战略复盘与调整 | 战略委员会 |

## 六、常见问题解答

### Q1：如何判断一个场景是否值得投入AI？
**A**：用"场景筛选矩阵"评估，优先选择"需求高频、数据成熟、容错空间大、易量化"的场景。避免在数据不成熟或容错率低的场景（如医疗诊断）直接上AI。

### Q2：业务部门不配合怎么办？
**A**：1）用"速赢"场景证明价值（如先做客服质检）；2）让业务部门参与场景选择；3）设定业务部门可量化的KPI与AI平台价值挂钩。

### Q3：技术团队想上"大模型"，但业务场景不清晰？
**A**：先让技术团队支持已明确的业务场景，不要为了"上大模型"而去找场景。技术服务于业务，不是业务服务于技术。
