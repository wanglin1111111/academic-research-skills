# 范式变革识别框架

## 一句话总述

> 渐进式创新在旧赛道上跑更快，范式变革开辟新赛道——区别在于，前者是在已有框架内优化参数，后者是重新定义框架本身。

## 四大范式转变模型

当前AI领域正在发生四条从"旧范式"到"新范式"的结构性转变：

### 转变1: 参数堆砌 → 结构定义智能

**旧范式**：更多数据 + 更大模型 + 更强算力 = 更智能
**新范式**：用结构/硬件创新降低对数据和算力的依赖

| 维度 | 旧范式 | 新范式 |
|------|--------|--------|
| 核心逻辑 | 暴力缩放 | 结构先行 |
| 关键资源 | 算力+数据 | 架构+机理 |
| 成本曲线 | 指数增长 | 线性可控 |
| 典型表现 | 更多参数更好效果 | 千元级硬件实现万级功能 |

**识别信号：**
- [ ] 行业是否陷入"谁参数多谁赢"的军备竞赛？
- [ ] 是否有技术路线能用结构创新绕过数据瓶颈？
- [ ] 端侧体验是否因云端依赖而受限？

**定位策略：** 如果行业在堆参数 → 寻找"用结构替代部分算法"的切入点

### 转变2: 通用AI → 物理世界AI

**旧范式**：文本/图像生成是AI的主战场
**新范式**：AI深入物理世界，解决因果推理和真实交互问题

| 维度 | 旧范式 | 新范式 |
|------|--------|--------|
| AI类型 | 信息生成 | 物理推理 |
| 数据来源 | 互联网文本 | 真实世界感知 |
| 价值定位 | 内容创作 | 工业决策 |
| 精度要求 | "差不多就行" | 工业级精度 |

**识别信号：**
- [ ] AI应用是否从"生成内容"走向"做出决策"？
- [ ] 行业是否需要因果推理而非统计相关性？
- [ ] 物理世界的数字化是否出现新数据源？

**定位策略：** 如果AI只停留在文本/图像 → 寻找"AI+物理世界"的交叉点

### 转变3: 工具替代 → 品类创造

**旧范式**：AI替代现有工具（AI写作替代写手、AI画图替代画师）
**新范式**：AI创造全新的产品品类（以前不存在的东西）

| 维度 | 旧范式 | 新范式 |
|------|--------|--------|
| 产品逻辑 | 更好的旧工具 | 全新的品类 |
| 竞争维度 | 效率/成本 | 存在性 |
| 定价权 | 被旧工具锚定 | 自由定价 |
| 市场规模 | 切割现有市场 | 创造新市场 |

**识别信号：**
- [ ] 产品是否在"替代XX"还是"创造XX"？
- [ ] 目标用户以前有没有类似产品？
- [ ] 产品是否定义了一个新品类名？

**定位策略：** 如果产品在替代旧工具 → 思考能否升维为"创造新品类"

### 转变4: 专家系统 → AI驱动研发

**旧范式**：AI辅助专家做重复工作
**新范式**：AI替代专业判断，重构知识生产方式

| 维度 | 旧范式 | 新范式 |
|------|--------|--------|
| AI角色 | 辅助工具 | 独立决策者 |
| 知识来源 | 人类专家 | 隐性知识蒸馏 |
| 价值层级 | 替代重复劳动 | 替代专业判断 |
| 可扩展性 | 受限于专家数量 | 随算力线性扩展 |

**识别信号：**
- [ ] 行业是否依赖少数专家的隐性知识？
- [ ] 这些隐性知识是否能被结构化并蒸馏为模型？
- [ ] AI是否能做出"专家级判断"而非"助手级操作"？

**定位策略：** 如果行业依赖专家 → 寻找"将隐性知识蒸馏为模型"的机会

## 范式变革识别工具

### 工具1: 范式定位雷达

对任何项目，从四个维度评估其范式位置：

`
维度1: 数据依赖度 (高依赖旧范式 ←→ 低依赖新范式)
维度2: 物理世界深度 (纯信息旧范式 ←→ 物理因果新范式)
维度3: 产品类型 (工具替代旧范式 ←→ 品类创造新范式)
维度4: 知识层级 (辅助操作旧范式 ←→ 专业判断新范式)
`

**📋 项目范式评估表：**

| 维度 | 得分(1-5) | 说明 |
|------|----------|------|
| 数据依赖度（1=重数据，5=轻数据） | ___ | __________ |
| 物理世界深度（1=纯信息，5=物理因果） | ___ | __________ |
| 产品类型（1=工具替代，5=品类创造） | ___ | __________ |
| 知识层级（1=辅助操作，5=专业判断） | ___ | __________ |
| **总分** | ___/20 | |

- **15-20**: 强范式变革者，新赛道定义者
- **10-14**: 混合型，有范式创新但未完全跳离旧框架
- **5-9**: 渐进式创新，在旧赛道上优化

### 工具2: 范式拐点信号清单

**技术拐点信号：**
- [ ] 某技术路线的成本骤降（如灵巧手从万元级到千元级）
- [ ] 某技术指标突破"可用拐点"（如AI物理模型达到工业级精度）
- [ ] 出现"反共识"的技术路线（如用结构替代算法）

**市场拐点信号：**
- [ ] 早期用户自发传播/命名产品（如用户自发命名新品类）
- [ ] 产品预售数据远超预期
- [ ] 出现"从未有过这种东西"的用户反馈

**生态拐点信号：**
- [ ] 大公司开始布局该方向
- [ ] 出现专门的投资主题/赛道
- [ ] 学术界开始收敛到某条技术路线

## 创业定位策略矩阵

根据你所在行业的范式成熟度，选择不同的进入策略：

| 阶段 | 范式已确立 | 范式未确立 |
|------|-----------|-----------|
| 你有核心技术 | 定义新品类，占位 | 赌一条路线，押注范式 |
| 你有场景优势 | 深耕场景，做最佳实践 | 从场景长出基础设施 |
| 你两者都有 | 范式变革的领导者 | 定义范式本身 |

**关键洞察：** 范式未确立时，场景优势比技术优势更重要——因为你从场景中长出来的解决方案，会自然定义新的范式。
