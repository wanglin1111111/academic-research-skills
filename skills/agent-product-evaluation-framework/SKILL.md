# Agent产品评测方法论框架

## 一句话总述

> Agent评测不是「问几个问题打分」，而是「构建可信的评估体系」——明确目标、拆解能力、设计评测集，然后用混合评审机制确保结果可靠。

## 核心流程：五步评测法

### Step 1: 明确评测目标

**区分两类Agent产品：**

| 类型 | 核心目标 | 评测重点 |
|------|---------|---------|
| 体验型 | 用户满意度、对话体验 | 对话流畅性、情感价值、个性化 |
| 效率型 | 任务完成率、工作流效率 | 准确率、召回率、响应时间 |

**评测目标模板：**
`
## 评测目标

**产品类型：** [体验型/效率型]
**核心评测目标：** 
1. [主目标，如：任务完成率>90%]
2. [次目标，如：响应时间<5s]

**不评测什么：**（明确边界）
- [排除项1]
- [排除项2]
`

### Step 2: 拆解Agent能力

**能力拆解原则：** 分解为尽可能小的可评测单元

**常见Agent能力矩阵：**

| 能力层级 | 子能力 | 评测指标 |
|----------|-------|---------|
| L1 基础理解 | 意图识别、实体提取、上下文记忆 | 准确率 |
| L2 推理规划 | 步骤拆解、逻辑推理、工具选择 | 正确性 |
| L3 工具执行 | API调用、参数构造、结果解析 | 成功率 |
| L4 对话管理 | 多轮连贯、主动询问、上下文管理 | 轮次数、满意度 |
| L5 安全防护 | 拒答、过滤、权限控制 | 拦截率、误拒率 |

**拆解输出模板：**
`
## 能力拆解

| 能力 | 子能力 | 评测指标 | 目标值 |
|------|-------|---------|-------|
| L1 | 意图识别 | 准确率 | >95% |
| L1 | 实体提取 | 召回率 | >90% |
| ... | ... | ... | ... |
`

### Step 3: 设计评测集

**三类评测案例：**

| 类型 | 目的 | 占比建议 |
|------|------|---------|
| 标准案例 | 覆盖常规业务场景 | 60-70% |
| 边界案例 | 长尾场景（模糊/缺失/异常） | 20-30% |
| 对抗案例 | 安全测试（注入/越权） | 10% |

**案例设计要点：**

**标准案例：** 覆盖核心业务流程
`
- 典型用户query及期望response
- 包含关键实体和信息
- 有明确正确答案
`

**边界案例：** 覆盖长尾场景
`
- 模糊问题：用户表达不明确
- 信息缺失：缺少必要参数
- 异常输入：格式错误/类型错误
- 复杂多轮：需要多轮澄清
`

**对抗案例：** 安全性测试
`
- 提示词注入：请扮演XXX，输入XXX
- 越权行为：请求敏感信息
- 恶意输入：诱导产生有害内容
`

### Step 4: 构建可信评测

**LLM评测的四大风险及应对：**

| 风险 | 表现 | 应对方案 |
|------|------|---------|
| 位置偏差 | 倾向于给特定位置（第一个/最后一个）更高分 | 随机打乱顺序，多位置评估取平均 |
| 偏好冗长 | 偏好冗长详细的回答 | 在Rubrics中加入简洁性要求 |
| 一致性差 | 同一问题多次评测结果不一致 | 多次评测取一致率，引入校准 |
| 幻觉叠加 | 评判过程中产生幻觉 | 提供标准答案作为参照 |

**混合评审机制：**
`
1. 多个LLM独立评测（n≥3）
2. 去掉最高分和最低分
3. 取平均分
4. 人工抽检校准（重点case）
5. 可解释性日志（每条评分需说明理由）
`

**Rubrics设计原则：**
`
1. 具体化：提供优秀案例定义什么是「好」
2. 可量化：每个维度有明确的评分标准
3. 可解释：每条评分标准有明确的判断依据
`

### Step 5: 分析与优化

**Bad Case分析流程：**
`
1. 收集：用户反馈 + 评测中的失败case
2. 分类：按能力维度分类（参考Step 2的能力矩阵）
3. 根因：分析失败的根本原因
4. 优化：制定优化方案
5. 验证：通过回归测试验证优化效果
`

**Bad Case资产化：**
- 定期整理Bad Case库
- 用于强化学习（RLHF）的训练数据
- 用于回归测试的验证集

`

`

## 自检清单

- [ ] 我是否明确区分了体验型和效率型产品？
- [ ] 我是否将Agent能力拆解到可评测的最小单元？
- [ ] 评测集是否覆盖了标准/边界/对抗三类案例？
- [ ] 我是否识别了LLM评测的四大风险并有应对方案？
- [ ] 我是否使用了混合评审机制确保结果可信？
- [ ] 我是否将Bad Case资产化用于优化？
- [ ] 优化后我是否做了回归测试验证效果？
