# AI创作提示词工具包

> 配套技能 `ai-native-creation-mindset`（思维层）
> 本技能提供**可直接使用的提示词模板**（执行层）

---

## 使用说明

每个模板包含：
- **适用场景**：什么时候用这个模板
- **完整提示词**：直接复制给AI使用
- **预期输出**：你会得到什么
- **迭代指引**：拿到输出后下一步做什么

**重要**：所有提示词中的 `[括号内容]` 都需要你替换成自己的实际情况。

## 模板一：认知深度自检

### 适用场景
你有一个创作/项目方向，但不确定自己的认知深度是否足够支撑AI放大。

### 完整提示词

```
请你帮我做一次"认知深度自检"。

我的背景：[你的专业/经验/兴趣领域]
我想做的方向：[简单描述你想做的东西]
我的目标用户：[谁会使用/消费这个作品]

请你按以下5个层次，分别问我问题，帮我判断我在该方向上的认知深度：

L1 — 我知道这个工具/领域的基本概念
L2 — 我理解能力边界，知道什么能做、什么做不好
L3 — 我能用现有工具实现脑子里已有的想法
L4 — 我能用工具探索脑子里还不确定的想法
L5 — 我能设计工具的用法，甚至让工具为我定制新工具

每个层次请你：
1. 给我出一个具体的"测试题"（不是选择题，是开放式问题）
2. 根据我的回答，告诉我达到了哪个层次
3. 如果某一层次未达到，给出具体的提升建议

最后，告诉我：如果我现在用AI来辅助这个方向，我的认知深度会让AI的输出达到什么水平？我应该怎么弥补认知短板？
```

### 预期输出
- 5个层次的测试题 + 你的回答评估
- 当前认知层次定位（L1-L5）
- 认知短板清单
- 具体弥补建议

### 迭代指引
拿到结果后：
- 如果认知在L1-L2：先不要急着用AI生成内容，先用AI帮你**学习该领域**
- 如果认知在L3-L4：可以直接用AI辅助创作，但要主动让AI帮你**探索不确定方向**
- 如果认知在L5：让AI帮你**定制专属工具**，进入工具创造者模式

## 模板二：枯萎技术组合器

### 适用场景
你想做一个项目，但不想用"大家都用的主流方案"，想找到差异化技术路线。

### 完整提示词

```
请你帮我用"枯萎技术的水平思考"方法，找到我的项目的差异化技术路线。

我的项目目标：[你想做什么，解决什么问题]
主流方案通常用的技术：[你知道的大家都在用的技术栈/工具]
我的约束条件：[预算/时间/技术能力限制]

请你帮我完成以下练习：

第一步：列出"枯萎技术清单"
帮我找出5-10种：成熟、低成本、稳定、但在这个领域被忽视的技术/工具/素材/数据源
（不要想"它们不够强"，想"它们有什么独特优势"）

第二步：强迫组合
从清单中随机选3-5项，强迫自己设计一个"用这些枯萎技术来实现我的目标"的方案
（越不可能的组合，越可能产生意外之喜）

第三步：评估可行性
对每个组合方案，评估：
- 技术可行性（1-10分）
- 差异化程度（1-10分，越高越不容易被复制）
- 成本优势（比主流方案低多少）

最后推荐：哪个组合最值得我深入探索？为什么？
```

### 预期输出
- 5-10个枯萎技术候选清单
- 3-5个强迫组合方案
- 每个方案的三维评估分数
- 最推荐方案 + 理由

### 迭代指引
拿到结果后：
1. 选最推荐的方案，让AI帮你做**技术可行性验证**（搜索是否有类似案例）
2. 如果方案看起来可行，进入模板三（工具定制）或模板四（MVP验证）

## 模板三：工具定制需求描述器

### 适用场景
你反复在做某些操作，想让AI帮你定制一个专属工具来简化流程。

### 完整提示词

```
请你帮我把自己变成"工具创造者"——帮我写出一份完整的工具定制需求描述，让AI（或开发者）能精准理解我想做什么。

我现在的创作流程：
[描述你目前的创作/工作流程，一步一步地描述]

我最频繁的重复操作：
1. [操作1，比如：每次都要手动调整格式]
2. [操作2，比如：每次都要在多个工具之间复制粘贴]
3. [操作3，比如：每次都要查同一个文档找参数]

我最痛苦的操作：
[描述最让你烦的操作，哪怕它出现频率不高]

我理想中的专属工具应该：
- 输入：[我希望给它什么输入]
- 输出：[我希望它给我什么输出]
- 特殊要求：[任何你觉得通用工具做不到的地方]

请你帮我：
1. 把以上需求整理成一份结构化的"工具定制需求文档"
2. 补充我可能没想到、但对这个工具很有用的功能点
3. 给出实现这个工具的三种路径（从"让AI写脚本"到"让AI搭完整系统"）
4. 推荐最可行的路径，并给出第一步的具体提示词

输出格式：可以直接复制给AI执行的需求文档。
```

### 预期输出
- 结构化工具定制需求文档
- 补充功能点清单
- 三种实现路径对比
- 推荐路径 + 第一步提示词

### 迭代指引
拿到需求文档后：
1. 直接把文档复制给AI，说"请按照这个需求，帮我实现"
2. 如果AI输出的工具不完整，用模板中的"补充功能点"继续迭代
3. 工具可用后，把它加入你的"专属工具库"，复用到未来项目

## 模板四：MVP极速验证

### 适用场景
你有一个模糊的想法，想在几小时内验证它是否值得深入。

### ⚠️ 重要：本模板已拆分为三个独立提示词

**原因**：原版一个提示词太长，AI容易漏掉部分内容。
**用法**：按顺序使用 4.1 → 4.2 → 4.3，每个阶段独立完成。

---

### 提示词 4.1（第一阶段：想法澄清）

```
请你帮我完成"MVP想法澄清"。

我的模糊想法：[用一句话描述你想做什么，不需要很清晰]

请你：
1. 问我5个关键问题，帮我把想法从"模糊"变成"具体"
2. 帮我定义：这个MVP最核心的体验是什么（只能保留一个）
3. 帮我确定：MVP不需要做什么（明确排除项，防止范围蔓延）

输出格式：
## 澄清后的想法
（具体、清晰的描述）

## 核心体验（唯一）
（只能保留一个，用一句话描述）

## 排除项清单
- ❌ 不需要______
- ❌ 不需要______
- ❌ 不需要______
```

**预期输出**：澄清后的想法 + 唯一核心体验 + 排除项清单

**下一步**：把输出复制到 4.2

---

### 提示词 4.2（第二阶段：MVP实现）

```
请你帮我生成这个MVP的可运行版本。

[粘贴 4.1 的完整输出]

首先，请你判断：这个MVP最适合用哪种形式实现？
- (a) 可运行代码（工具/脚本/自动化）
- (b) 原型描述 + 界面草图（产品/功能/界面）
- (c) 详细规格说明（内容/服务/方法论）

我选择：______（填字母）

请你按我选择的形式，生成MVP：

如果选 (a)：生成完整可运行代码（带注释）
如果选 (b)：生成原型描述 + ASCII界面草图 + 用户流程说明
如果选 (c)：生成详细规格说明（包含：目标用户、核心功能、差异化、 monetization 假设）

生成后，请你自己快速体验/审查一遍，指出：
1. 哪个部分最可能让用户感到"有意思"？
2. 哪个部分最可能需要改进？
```

**预期输出**：MVP实现（代码/原型/规格） + AI自我审查

**下一步**：把输出复制到 4.3

---

### 提示词 4.3（第三阶段：挑战者评估）

```
请你作为"严厉的挑战者"，帮我评估这个MVP是否值得深入。

[粘贴 4.2 的完整输出]

请你问我4个尖锐的问题（不是温和的，是真正挑战我的问题）：

1. 关于价值："为什么用户会用你的而不是已有的解决方案？你的差异化到底是什么？"
2. 关于壁垒："你的这个MVP，有什么是别人复制不了的？如果没有，你打算怎么建立壁垒？"
3. 关于执行："你真的会在未来3天内继续改进它吗？如果答案是'看情况'，说明你热情不够"
4. 关于市场："你愿意为这个MVP付多少钱？如果答案是0或接近0，说明它对你来说价值不够"

根据我的回答，请你帮我做决策：
- ✅ 继续深入（有清晰价值 + 有差异化 + 有执行动力）
- 🔄 换方向（价值不明确 或 差异化不足）
- 📦 暂时搁置（有价值但当前执行动力不足，记录原因）

最后，告诉我：如果继续深入，第一天应该做什么？
```

**预期输出**：4个尖锐问题 + 你的回答 + 决策（继续/换方向/搁置） + 第一天行动

### 迭代指引
- **继续深入**：进入模板三（工具定制）或模板六（反馈循环设计）
- **换方向**：回到模板二（枯萎技术组合），换一个技术路线
- **暂时搁置**：记录原因到 `memory/YYYY-MM-DD.md`，3个月后再评估

## 模板五：创作初心澄清

### 适用场景
你准备开始一个创作项目，想在开始之前先弄清楚"我到底为什么要做这个"。

### 完整提示词

```
请你帮我做一次"创作初心澄清"练习。

我想做的项目：[项目名称或简单描述]

请你用"灵魂三问"帮我深入探索：

第一问：如果这款作品完全失败、没有人喜欢，我还会想做吗？
- 请你问我追问题，帮我找到"即使失败也想做"的内在动机
- 如果我发现答案是"不会"，请你帮我重新找到真正的动机

第二问：我想让哪类人感受到什么？
- 请你帮我具体化"哪类人"（越细越好，最好是具体的人群画像）
- 请你帮我具体化"感受到什么"（是情感？认知？冲动？安慰？）
- 最终帮我写成一句："让_______的人感受到_______"

第三问：如果只能保留一个核心体验，那是什么？
- 请你帮我把所有"想要的功能/内容"列出来
- 然后一个个划掉，直到只剩一个
- 这个"唯一保留"的就是我的核心体验

最后，请你帮我写一段"创作初心声明"（100字以内），让我在后续创作中随时回头看，不迷失方向。
```

### 预期输出
- 灵魂三问的深度探索结果
- 具体化的人群画像 + 感受描述
- 唯一核心体验定义
- 100字以内的创作初心声明

### 迭代指引
- 把"创作初心声明"贴在 workspace 显眼位置
- 每次做创作决策时，问自己："这个决策符合我的初心吗？"
- 如果创作过程中感到迷失，重新做一遍这个练习

## 模板六：反馈循环设计器

### 适用场景
你的创作流程反馈太慢（超过1天才能看到结果），想重新设计成极速反馈循环。

### 完整提示词

```
请你帮我重新设计我的创作反馈循环，把"从想法到看到结果"的周期从[当前周期]压缩到[目标周期]以内。

我当前的创作流程：
[详细描述你现在的创作流程，每一步需要多久]

我认为最慢的瓶颈环节：
[描述哪个环节最慢，为什么]

请你帮我：

第一步：找出所有"超过[目标周期]才能看到结果"的环节
对每个环节，分析：
- 为什么这么慢？
- 有没有办法让它变快？
- 如果变快会牺牲什么质量？

第二步：设计"极速反馈循环"方案
帮我重新设计流程，目标是：
- 任何环节不超过[目标周期]
- 如果某个环节实在无法加速，能不能把它拆成更小的步骤，每步都有反馈？

第三步：给出具体的AI辅助方案
告诉我：
- 在哪个环节，AI可以帮我加速？
- 我需要让AI生成什么，才能快速看到结果？
- 我怎么设置"快速测试版本" vs "最终质量版本"的两层标准？

输出格式：前后对比流程图 + 具体AI提示词（每个加速环节对应的提示词）。
```

### 预期输出
- 瓶颈环节分析报告
- 极速反馈循环设计方案（前后对比）
- 每个加速环节对应的AI提示词

### 迭代指引
- 先挑最慢的一个瓶颈环节，用对应的AI提示词加速
- 验证有效后，再逐步改造其他环节
- 每改造一个环节，记录"压缩前周期 vs 压缩后周期"

## 提示词使用技巧

### 技巧1：分批次给AI输入
不要一次性把整个提示词 + 所有背景都塞给AI。
先给提示词，等AI回复后，再逐步补充背景。

### 技巧2：要求结构化输出
在提示词最后加上：
```
请你用以下格式输出：
## 分析结果
## 具体建议
## 下一步行动
## 需要我补充的信息
```

### 技巧3：迭代而不是重来
如果AI的输出不够好：
- ❌ 不好：删掉重来，重新描述
- ✅ 好：在原有输出基础上说"第X部分请再详细一点，具体是……"
