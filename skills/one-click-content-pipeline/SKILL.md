# 一键内容生产管道（One-Click Content Pipeline）

> 把"从一段文字到可发布内容"的全流程封装成可复用工作流
> 
> 核心思想：Skill的本质是经验的具象化 —— 把专业的隐性知识封装成可点击的"开关"

---

## 核心理念

### 理念一：Skill封装重复性工作
**本质**：把"每次都要做"的事情，变成"点一下就好"的工具

**例子**：
- ❌ 每次做PPT都要调格式、找配图、检查对齐
- ✅ 封装成Skill：输入大纲 → 自动生成美观PPT（含配图、自动检查对齐）

### 理念二：全流程闭环
**目标**：从"一段文字"到"可发布内容"，中间不打开任何专业软件

**流程**：
```
输入：一段文字（想法/大纲/草稿）
  ↓
AI处理：自动生成图文/视频/PPT等
  ↓
输出：可直接发布的内容
```

### 理念三：上下文质量决定输出质量
**核心**：AI需要"高质量示例"才能输出好结果

**怎么做**：
1. 收集你领域内**最优质的内容**（3-5个示例）
2. 把这些示例作为"上下文"投喂给AI
3. AI会学习这些示例的风格、结构、质量标准

### 理念四：自然语言迭代
**核心**：不满意？直接用自然语言告诉AI，它快速响应

**例子**：
- ❌ 不好："重新生成"（太模糊）
- ✅ 好："别用深色遮罩，用浅色"、"标题字号太大，改小一点"

## 使用场景

### 场景一：自媒体图文创作
**需求**：根据一段文案，生成适合小红书/公众号的图文

**工作流**：
```
输入：一段文案（如"香港徒步攻略"）
  ↓
步骤1：AI分析文案结构，提取关键信息
  ↓
步骤2：AI生成3:4比例的图文组图（适合竖版平台）
  ↓
步骤3：AI自动配色、排版、添加装饰元素
  ↓
输出：可直接发布的图文组图（通常3-6张）
```

### 场景二：PPT快速生成
**需求**：根据大纲/文档，生成美观的PPT

**工作流**：
```
输入：大纲（Markdown格式）或文档（Word/PDF）
  ↓
步骤1：AI解析大纲，生成幻灯片结构
  ↓
步骤2：AI自动选择版式（标题页、内容页、图文页等）
  ↓
步骤3：AI自动配图（从图库或AI生成）
  ↓
步骤4：AI后验证（检查居中、对齐、内容溢出等美学问题）
  ↓
输出：HTML格式PPT（可在浏览器直接演示）
```

### 场景三：内容多平台适配
**需求**：同一段内容，适配不同平台（公众号/小红书/知乎/推特）

**工作流**：
```
输入：一段内容（如博客文章）
  ↓
步骤1：AI分析各平台特点（字数限制、图片比例、标题风格等）
  ↓
步骤2：AI生成适配各平台的版本
  ↓
步骤3：AI自动生成配图（不同比例）
  ↓
输出：各平台可直接发布的版本（含配图）
```

## 实战：从0到1创建你的内容生产管道

### 第一步：识别重复性工作
**问自己**：
1. 我每次做内容，哪些步骤是重复的？
2. 哪些操作让我觉得"又来了，好烦"？
3. 如果有一个工具能自动完成这些，我最希望它做什么？

**常见重复性工作**：
- 调格式（PPT/文档/图文）
- 找配图（每次都要去图库搜）
- 检查排版（对齐、居中、内容溢出）
- 多平台适配（同一内容发多个平台）
- 生成摘要/标题（从长文中提取）

### 第二步：把重复性工作封装成Skill
**方法**：用AI帮你生成Skill文件

**提示词模板**：
```
我想把一个重复性工作流程封装成Skill（可复用工具）。

我的重复性工作：
[描述你每次都要做的步骤，一步一步写]

我希望的输入：
[用户需要提供给AI什么？比如：一段文字、一个大纲、一张图片]

我希望的输出：
[AI应该生成什么？比如：一套图文、一个PPT、一个视频脚本]

请你帮我：
1. 把这个流程拆解成清晰的步骤
2. 生成一个Skill文件（SKILL.md），包含：
   - 触发场景（什么时候用这个Skill）
   - 触发词（用户说什么话会触发这个Skill）
   - 完整工作流（每一步AI应该做什么）
   - 质量标准（怎么判断输出是好的）
3. 生成一个测试案例，让我能直接测试这个Skill
```

### 第三步：投喂高质量上下文
**核心**：AI需要"看到好例子"才能输出好结果

**做法**：
1. 收集你领域内**最优质的3-5个内容**（可以是别人的，也可以是你自己最好的作品）
2. 把这些内容作为"示例"放进Skill文件（或放在单独的`examples/`目录）
3. 在Skill文件中告诉AI："请参考这些示例的风格/结构/质量标准"

**例子**（PPT Skill）：
```markdown
## 高质量示例

请参考以下PPT示例的风格：
- 示例1：[标题] 简洁有力，不超过10个字
- 示例2：[配色] 使用渐变色，避免纯色背景
- 示例3：[排版] 左文右图，留白充足
```

### 第四步：测试与迭代
**测试**：用真实内容测试你的Skill，看输出是否符合预期

**迭代**：不满意？直接用自然语言告诉AI
- ✅ 好："配色太暗，改亮一点"
- ✅ 好："第三页的图表太复杂，简化一下"
- ✅ 好："整体风格不对，参考示例2重新生成"

**记录**：把每次迭代的"修改指令"记录下来，后续可以自动化

- 你的平台运营策略

## 安全与可控性

### 高风险操作需用户批准
**原则**：涉及文件读写、网络请求、执行命令等操作，必须让用户二次确认

**例子**：
- ❌ 不好：AI直接删除本地文件（太危险）
- ✅ 好：AI提示"我将删除文件XXX，是否继续？"，用户点击"是"才执行

### 本地文件处理在安全沙箱内运行
**原则**：AI处理的文件应该在隔离环境中，避免泄露隐私

**做法**：
- 使用沙箱环境（如Docker容器、虚拟机）
- 处理完后自动清理临时文件
- 不把敏感文件上传到云端

## 实战案例：香港徒步攻略图文生成

### 需求
输入一段文案："香港徒步攻略：麦理浩径第一段，难度中等，风景优美，适合周末出行"
输出适合小红书的图文组图（3:4比例，3-6张）

### 工作流（通过Skill自动化）
```
步骤1：AI解析文案，提取关键信息
  - 地点：香港麦理浩径第一段
  - 难度：中等
  - 特点：风景优美
  - 适合：周末出行

步骤2：AI生成图文结构
  - 第1张：封面（标题 + 主图）
  - 第2张：路线概览（地图 + 关键信息）
  - 第3张：难度说明（配图 + 文字）
  - 第4张：风景亮点（多图拼接）
  - 第5张：实用信息（交通、装备、注意事项）

步骤3：AI自动配图
  - 方案A：从图库搜索"香港徒步"相关图片
  - 方案B：AI生成示意图（如路线地图、风景插画）

步骤4：AI自动排版
  - 比例：3:4（适合小红书）
  - 配色：清新自然风格（绿色、蓝色）
  - 字体：简洁无衬线字体
  - 装饰：添加图标、分隔线、标签等

步骤5：后验证
  - 检查文字是否超出边界
  - 检查图片是否清晰（分辨率足够）
  - 检查整体风格是否统一

输出：5张图文（JPG格式，1080x1440像素）
```

### 实际使用（通过AI对话）
**用户输入**：
```
请根据以下文案生成小红书图文：
"香港徒步攻略：麦理浩径第一段，难度中等，风景优美，适合周末出行"

请使用我的"社交媒体图文生成Skill"
```

**AI输出**：
```

[图片2] 路线概览.png
[图片3] 难度说明.png
[图片4] 风景亮点.png
[图片5] 实用信息.png

如需修改，请直接告诉我，例如：
- "配色太暗，改亮一点"
```

## 常见问题

### Q1：我没有编程基础，能创建Skill吗？
**A**：可以！Skill文件（SKILL.md）是**Markdown格式**，就像写文档一样。AI会帮你生成，你只需要描述清楚你的需求。

### Q2：Skill生成的内容质量不够高怎么办？
**A**：检查以下3点：
1. **上下文质量** — 你投喂的示例够不够好？
2. **指令清晰度** — 你的提示词够不够具体？
3. **迭代次数** — 第一次输出往往不够好，需要2-3轮迭代

### Q3：如何让Skill"记住"我的偏好？
**A**：在Skill文件中添加一个"用户偏好"章节，例如：
```markdown
## 用户偏好

- 配色：清新自然风格（绿色、蓝色）
- 字体：简洁无衬线字体
- 排版：左文右图，留白充足
- 语气：亲切自然，像朋友聊天
```

### Q4：Skill可以分享给其他人吗？
**A**：可以！Skill文件（SKILL.md）是纯文本，可以直接分享。其他人只需要把文件放到OpenClaw的skills目录，就能使用你的Skill。
