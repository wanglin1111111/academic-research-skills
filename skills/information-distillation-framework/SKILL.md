# 信息蒸馏框架

## 一句话总述

> 好的提炼不是"压缩"，而是"蒸馏"——压缩只减小体积，蒸馏改变纯度。目标是从原始信息中提取出决策级、行动级的高纯度精华。

## 核心模型：四层蒸馏

### Layer 1: 信号筛选（去噪）

**目标：** 从原始信息中分离信号与噪音

| 信号类型 | 识别方法 | 保留？ |
|----------|---------|--------|
| 核心结论 | "关键是/最重要的是/结论是" | ✅ 必留 |
| 支撑证据 | 数据、案例、对比 | ✅ 选关键 |
| 过程描述 | "我们讨论了…然后又讨论了…" | ❌ 跳过 |
| 重复信息 | 同一观点的多次表述 | ❌ 合并 |
| 离题内容 | 与目标无关的讨论 | ❌ 跳过 |

**筛选原则：** 以"读完后能做什么决定"为标准——对决策无帮助的信息就是噪音。

### Layer 2: 结构识别（建骨架）

**目标：** 识别信息的内在逻辑结构

**五种常见信息结构：**

| 结构类型 | 特征 | 蒸馏方式 |
|----------|------|----------|
| 问题-方案 | 先描述问题，再给出方案 | 问题1行+方案1行 |
| 观点-论据 | 先亮观点，再铺论据 | 观点+top3论据 |
| 时间线 | 按时间顺序叙述 | 提取关键节点和转折点 |
| 分类列举 | 多个平行的类别 | 每类1行总结 |
| 因果链 | A导致B导致C | 保留因果链，砍掉中间过程 |

**识别方法：** 快速扫描，找到"因此/所以/关键是"等转折词，它们指向核心结构。

### Layer 3: 逻辑提炼（蒸馏精华）

**目标：** 将信息提炼为"结论-证据-行动"三段式

**三段式规范：**

`
📌 结论：1-2句话的核心判断（管理者级）
📊 证据：3-5条支撑性事实/数据（分析师级）
✅ 行动：明确任务+负责人+截止日（执行者级）
`

**提炼规则：**
- 结论必须是判断句（"应该/需要/将"），不能是描述句（"讨论了/提到了"）
- 证据必须可验证（数据>案例>观点引用）
- 行动必须SMART化（具体/可衡量/可执行/相关/有时限）

### Layer 4: 输出适配（定向交付）

**目标：** 根据读者角色调整输出粒度

| 读者类型 | 需要的层级 | 输出格式 |
|----------|-----------|---------|
| 决策者 | 结论+行动 | 1段话+行动表 |
| 执行者 | 行动+证据 | 行动表+关键背景 |
| 分析者 | 全部三层 | 完整三段式 |
| 存档 | 原始信息+结构化摘要 | 摘要前置+附录 |

## 结构化思维框架

### 金字塔原则（结论先行）

`
        [核心结论]
       /    |    \
  [论据1] [论据2] [论据3]
   / \      |      |
 [事实]  [事实]   [事实]
`

**应用规则：**
1. 任何输出必须从结论开始
2. 每个结论有≤5个支撑论据
3. 每个论据有1-3个事实支撑
4. 同层论据之间必须MECE（互斥且完全穷尽）

### SCQA框架（故事线构建）

- **S (Situation)**：背景情境——"我们现在的状态"
- **C (Complication)**：冲突/问题——"出了什么问题"
- **Q (Question)**：核心问题——"需要解决什么"
- **A (Answer)**：答案/方案——"我们的方案是"

**适用场景：** 需要说服他人时（汇报、提案、建议书），先用SCQA讲"为什么"，再用金字塔讲"怎么做"。

## "黄金1小时"方法论

> 信息蒸馏有时效性——时间越久，上下文记忆越模糊，提炼质量越低。

**时效性衰减曲线：**

| 时间 | 记忆保留 | 蒸馏质量 | 建议 |
|------|---------|---------|------|
| 0-1小时 | 90%+ | 最高 | 立即蒸馏 |
| 1-4小时 | 70% | 良好 | 当天完成 |
| 4-24小时 | 50% | 可用 | 需回听/回看 |
| 24h+ | <30% | 低 | 必须回听/回看 |

**黄金1小时操作流程：**

`
0-15分钟：快速回顾笔记，标注"结论词"（因此/所以/关键/决定）
15-30分钟：按四层蒸馏模型筛选和结构化
30-45分钟：写出三段式输出
45-60分钟：校对准确性，补充遗漏的行动项
`

## 蒸馏质量自检

**5问检验法：**
1. 结论是判断句还是描述句？（必须是判断句）
2. 证据能独立支撑结论吗？（不能则需要补充）
3. 行动项是否SMART化？（每项有负责人+截止日）
4. 删掉任何一条信息，结论还成立吗？（成立说明该条冗余）
5. 给5分钟让别人读完，ta能复述核心结论吗？（不能则太长或太模糊）

## 自检清单

- [ ] 我是否按四层蒸馏模型筛选了信息？
- [ ] 我的输出是否是"结论-证据-行动"三段式？
- [ ] 我的结论是否是判断句？
- [ ] 我的行动项是否SMART化？
- [ ] 我是否在"黄金1小时"内完成蒸馏？
- [ ] 我是否根据读者角色调整了输出粒度？
