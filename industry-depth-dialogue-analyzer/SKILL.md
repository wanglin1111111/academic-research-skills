# 行业深度对话分析框架

## 概述

本技能提供系统化的方法论，用于分析任何行业的深度对话内容（圆桌会议、专家访谈、行业辩论等），并提取可操作的市场洞察和趋势预测。

核心理念：从多角度对话中识别**共识**、**分歧**和**盲点**，构建行业发展的立体画像。

## 核心分析框架

### 第一步：识别参与视角

任何深度行业对话都包含多个专业视角。常见视角类型：

- **投资视角**：资本逻辑、估值逻辑、投资回报、风险判断
- **技术视角**：技术路线、研发重点、技术瓶颈、创新方向
- **产品/应用视角**：场景落地、用户需求、产品迭代、商业化路径
- **市场/商业视角**：市场规模、竞争格局、商业模式、盈利路径
- **政策/监管视角**：政策影响、监管要求、合规成本、政策红利
- **供应链/产业视角**：产业链结构、供应瓶颈、成本结构、产能布局

**操作指南**：阅读对话时，先标注每位发言者的核心视角，构建"视角-观点"矩阵。

### 第二步：提取核心议题

从对话中识别反复出现、引发争议或达成共识的核心议题。使用以下模板记录：

```markdown
### 议题：[议题名称]

**争议点**：
- 观点A：[描述] (支持者：...)
- 观点B：[描述] (支持者：...)

**共识**：
- [达成的共识1]
- [达成的共识2]

**未知/盲点**：
- [尚未厘清的问题]
```

### 第三步：识别核心资源与瓶颈

每个行业都有其关键资源约束。从对话中识别：

1. **数据资源**：数据获取、数据质量、数据成本
2. **算力资源**：计算能力、算力成本、算力分配
3. **人才资源**：关键技能人才、人才培养周期
4. **资本资源**：融资环境、投资热度、资本回报周期
5. **供应链资源**：核心零部件、产能、供应商议价能力
6. **政策资源**：牌照、资质、政府支持

**分析方法**：对于每种资源，记录：
- 当前供给状况
- 未来需求预测
- 成本结构变化趋势
- 瓶颈突破路径

### 第四步：竞争格局分析

从对话中提炼竞争格局的关键维度：

**市场参与者分类**：
- 巨头/大厂：战略意图、资源优势、劣势
- 创业公司：聚焦领域、差异化、生存策略
- 垂直玩家：细分领域、护城河、成长空间

**竞争维度**：
- 技术路线竞争
- 场景落地竞争
- 人才争夺
- 资本博弈
- 供应链整合

**生存法则**：
- 创业公司如何与巨头共存？
- 垂直整合 vs 生态合作？
- 先发优势 vs 后发制人？

### 第五步：未来趋势预测

基于对话中的观点，构建未来12-24个月的预测框架：

**预测维度**：
1. **技术突破**：哪些技术会在什么时间窗口成熟？
2. **商业化落地**：哪些场景会率先实现商业闭环？
3. **市场格局**：市场份额会如何变化？淘汰赛何时开始？
4. **成本下降曲线**：核心成本项会如何下降？
5. **监管政策**：政策会如何演变？

**预测模板**：
```markdown
### 预测：[预测内容]

**时间窗口**：[12个月/24个月]
**关键前提**：[实现该预测的必要条件]
**验证指标**：[如何验证预测是否成立]
**影响分析**：[对行业格局的影响]
```

## 输出成果结构

完成分析后，生成结构化报告：

```markdown
# [行业名称]深度分析报告

## 一、对话概况
- 时间、参与者、核心议题

## 二、多视角观点汇总
- 各视角的核心观点和逻辑

## 三、核心议题深度剖析
- 争议议题、共识议题、未知议题

## 四、资源与瓶颈分析
- 关键资源供需状况、成本结构、突破路径

## 五、竞争格局研判
- 市场参与者、竞争维度、生存法则

## 六、未来趋势预测
- 12个月预测、24个月预测、验证指标

## 七、投资/战略建议
- 基于分析的可行动方案
```

## 使用建议

1. **适用性广**：本框架适用于任何新兴技术或快速演变的行业
2. **举一反三**：从一个行业对话中提取的分析逻辑，可迁移到类似行业
3. **持续迭代**：行业对话是动态的，定期更新分析以保持洞察的时效性
4. **交叉验证**：单一对话可能有偏见，结合多场对话进行交叉验证
